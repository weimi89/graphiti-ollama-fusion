# 使用 Graphiti MCP 工具的指令

> **此為核心指令，必須嚴格遵守，不可跳過任何步驟**

## 專案識別（強制）

**自動取得專案名稱：** 從當前工作目錄的最後一層目錄名稱作為專案 group_id。這是強制規則，不可使用其他命名方式。

**專案記憶隔離：** 每個專案必須使用獨立的 group_id，絕對不可將不同專案的記憶混在一起。

**全域記憶共享：** 只有跨專案通用的偏好和程式才能使用 `global` 作為 group_id。

## 開始任何任務之前（必執行）

**這是強制步驟，每次開始任務前必須執行，不可跳過：**

1. **必須搜索專案記憶：** 使用 `search_memory_nodes` 搜索當前專案的 group_id
2. **必須搜索全域記憶：** 使用 `search_memory_nodes` 搜索 `global`
3. **必須搜索事實：** 使用 `search_memory_facts` 發現相關的關係和事實資訊
4. **必須審查結果：** 仔細檢查所有匹配的偏好、程式和事實

**按實體類型過濾：** 在節點搜索中指定 "Preference"（偏好）或 "Procedure"（程式）以獲得針對性結果。

## 工作過程中（必須遵守）

**遵循發現的偏好：** 發現任何偏好後，必須使工作與其保持一致，不可忽略。

**嚴格按照程式執行：** 如果找到適用的程式，必須嚴格按步驟執行，不可跳過或修改步驟。

**應用相關事實：** 必須使用事實資訊來指導決策和建議。

**保持一致性：** 必須與先前識別的偏好、程式和事實保持一致，不可產生矛盾。

## 儲存新的或更新的資訊（必執行）

**當出現以下情況時，必須立即儲存，不可延遲或忘記：**

- 用戶表達任何需求或偏好 → 立即使用 `add_memory_simple` 儲存
- 發現用戶希望如何完成某操作 → 記錄為程式
- 了解到實體之間的連接 → 儲存為事實
- 完成重要任務或解決問題 → 記錄成果和學習

**使用正確的 group_id（強制）：**
- 專案特定內容 → 必須使用當前專案 group_id
- 跨專案通用內容 → 必須使用 `global`

## 觸發規則（自動執行）

當看到以下關鍵詞時，必須立即執行對應動作：

- "開始", "任務", "請幫我", "參考" → `search_memory_nodes`（專案 + 全域）+ `search_memory_facts`
- "完成", "結束", "好了" → `add_memory_simple` 記錄成果（專案 group_id）
- "測試", "檔案組織" → 記住：所有測試必須放在 `tests/` 目錄
- "偏好", "程式", "做法" → 搜索並嚴格遵循既有設定
- "全域", "所有專案" → 使用 `global` 作為 group_id

## group_id 命名規範

**必須按照以下格式命名，不可自創格式：**

- **專案主記憶：** `<專案名稱>`
- **專案偏好：** `<專案名稱>:preferences`
- **專案流程：** `<專案名稱>:procedures`
- **專案決策：** `<專案名稱>:decisions`
- **專案錯誤：** `<專案名稱>:errors`
- **全域通用：** `global`
- **全域偏好：** `global:preferences`

## 最佳實踐

**建議前先搜索：** 在提出任何建議之前，必須先檢查是否存在既定知識。

**結合節點和事實搜索：** 對於複雜任務，必須同時搜索節點和事實以構建完整圖景。

**使用 center_node_uuid：** 在探索相關資訊時，圍繞特定節點進行搜索。

**優先考慮具體匹配：** 更具體的資訊優先於一般資訊。

**主動識別模式：** 如果注意到使用者行為中的模式，應主動儲存為偏好或程式。

**重要提醒：** 知識圖譜是您的記憶。必須持續使用它來提供個性化協助，必須尊重用戶既定的程式和事實背景。
