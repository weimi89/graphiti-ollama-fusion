# 使用 Graphiti MCP 工具的指令

## 開始任何任務之前

**始終先搜索：** 在開始工作之前，使用 `search_memory_nodes` 工具查詢相關的偏好設定和程式。

**同時搜索事實：** 使用 `search_memory_facts` 工具發現可能與您的任務相關的關係和事實資訊。

**按實體型別過濾：** 在節點搜索中指定 "Preference"（偏好）或 "Procedure"（程式）以獲得針對性結果。

**審查所有匹配項：** 仔細檢查與目前任務匹配的任何偏好、程式或事實。

## 始終儲存新的或更新的資訊

**立即捕獲需求和偏好：** 當用戶表達需求或偏好時，立即使用 `add_memory_simple` 儲存它。長文本會自動切分為多段處理。

**明確標識更新：** 如果某些內容是對現有知識的更新，請明確說明。

**清晰記錄程式：** 當您發現使用者希望如何完成某些操作時，將其記錄為程式。

**記錄事實關係：** 當您瞭解到實體之間的連線時，將這些資訊儲存為事實。

**明確分類：** 為偏好和程式標註清晰的類別，以便日後更好地檢索。

## 工作過程中

**遵循發現的偏好：** 使您的工作與找到的任何偏好保持一致。

**嚴格按照程式執行：** 如果找到適用於目前任務的程式，請嚴格按步驟執行。

**應用相關事實：** 使用事實資訊來指導您的決策和建議。

**保持一致性：** 與先前識別的偏好、程式和事實保持一致。

## 最佳實踐

**建議前先搜索：** 在提出建議之前，始終檢查是否存在既定知識。

**結合節點和事實搜索：** 對於複雜任務，同時搜索節點和事實以構建完整圖景。

**使用 center_node_uuid：** 在探索相關資訊時，圍繞特定節點進行搜索。

**優先考慮具體匹配：** 更具體的資訊優先於一般資訊。

**主動識別模式：** 如果您注意到使用者行為中的模式，考慮將其儲存為偏好或程式。

**長文本使用背景模式：** 對於較長的記憶內容，使用 `background=true` 避免阻塞等待。

**重要提醒：** 知識圖譜是您的記憶。持續使用它來提供個性化協助，尊重使用者既定的程式和事實背景。

## 可用的 MCP 工具（11 個）

### 記憶管理工具

#### `add_memory_simple` - 添加記憶到知識圖譜

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `name` | string | Y | - | 記憶片段的名稱 |
| `episode_body` | string | Y | - | 記憶內容（超過 800 字元自動切分） |
| `group_id` | string | | "default" | 記憶分組 ID（建議按專案隔離） |
| `source_description` | string | | "MCP Server" | 來源描述 |
| `source` | string | | "text" | 內容類型：text、json、message |
| `episode_uuid` | string | | null | 自訂 UUID（用於更新現有記憶） |
| `use_safe_mode` | boolean | | false | 安全模式（跳過實體提取，快但不可搜尋） |
| `background` | boolean | | false | 背景處理（立即返回 task_id，適合長文本） |
| `excluded_entity_types` | array | | null | 排除的實體類型（減少不需要的提取量） |

**兩種模式：**
- **完整模式（預設）**：`use_safe_mode=false`，執行完整的實體提取和關係建立，記憶可被 `search_memory_nodes` 和 `search_memory_facts` 搜尋。處理時間較長（視文本長度和模型速度而定）。
- **安全模式**：`use_safe_mode=true`，僅建立 EpisodicNode，秒速完成但記憶**無法**被搜尋工具找到。

**效能提示：**
- 短文本（<800 字元）：直接處理，約 30-40 秒
- 長文本（>800 字元）：自動切分為多段，每段獨立處理
- 使用 `background=true` 可避免 MCP 呼叫阻塞，透過 `get_memory_task_status` 追蹤進度
- 切分閾值和段落大小可透過環境變數 `GRAPHITI_CHUNK_THRESHOLD`、`GRAPHITI_MAX_CHUNK_SIZE` 調整

#### `search_memory_nodes` - 搜尋記憶節點（實體）

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `query` | string | Y | - | 搜尋關鍵字（自然語言） |
| `max_nodes` | integer | | 10 | 返回節點的最大數量 |
| `group_ids` | array | | null | 用於篩選的分組 ID 列表 |
| `entity_types` | array | | null | 實體類型過濾（如 "Preference"、"Procedure"） |

> **注意**：僅能搜尋以完整模式（`use_safe_mode=false`）添加的記憶。安全模式添加的記憶不會產生可搜尋的實體節點。

#### `search_memory_facts` - 搜尋記憶事實（關係）

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `query` | string | Y | - | 搜尋關鍵字 |
| `max_facts` | integer | | 10 | 返回事實的最大數量 |
| `group_ids` | array | | null | 用於篩選的分組 ID 列表 |
| `center_node_uuid` | string | | null | 圍繞特定節點搜索相關事實 |

#### `get_episodes` - 獲取最近的記憶片段

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `last_n` | integer | | 10 | 獲取最近記憶片段的數量 |
| `group_id` | string | | "" | 用於篩選的分組 ID |

---

### 任務管理工具

#### `get_memory_task_status` - 查詢背景任務進度

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `task_id` | string | Y | - | 背景任務 ID（由 `add_memory_simple(background=true)` 返回） |

**返回資訊：**
- `task_id` - 任務 ID
- `status` - 狀態：pending / processing / completed / failed
- `name` - 記憶名稱
- `group_id` - 分組 ID
- `chunks_total` - 總切分段數
- `chunks_done` - 已完成段數
- `created_at` - 建立時間
- `completed_at` - 完成時間
- `result` - 處理結果（完成時）
- `error` - 錯誤訊息（失敗時）

---

### 刪除與查詢工具

#### `delete_episode` - 刪除記憶片段

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `uuid` | string | Y | - | 要刪除的記憶片段 UUID |

#### `delete_entity_edge` - 刪除實體邊（關係）

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `uuid` | string | Y | - | 要刪除的實體邊 UUID |

#### `get_entity_edge` - 獲取實體邊詳細資訊

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `uuid` | string | Y | - | 要查詢的實體邊 UUID |

**返回資訊包含：**
- `source_node_uuid` - 來源節點 UUID
- `target_node_uuid` - 目標節點 UUID
- `fact` - 事實描述
- `fact_embedding` - 向量嵌入（如有）
- `episodes` - 相關記憶片段列表
- `created_at` - 建立時間
- `expired_at` - 過期時間（如有）
- `valid_at` - 生效時間
- `invalid_at` - 失效時間（如有）

---

### 系統工具

#### `test_connection` - 測試連接狀態

檢查各組件的連接狀態，無需參數。

**返回資訊：**
- Neo4j 資料庫連接狀態
- Ollama LLM 服務狀態（含模型名稱）
- 嵌入器服務狀態（含模型名稱）

#### `get_status` - 獲取服務狀態

獲取完整的服務狀態和資料庫統計，無需參數。

**返回範例：**
```json
{
  "status": "healthy",
  "service": "graphiti-ollama-mcp",
  "components": {
    "database": "connected",
    "llm": "connected",
    "embedder": "connected"
  },
  "group_count": 5,
  "episode_count": 150
}
```

#### `clear_graph` - 清除圖資料庫

| 參數 | 類型 | 必填 | 預設值 | 說明 |
|------|------|------|--------|------|
| `group_ids` | array | | null | 僅清除指定分組（留空則清除全部） |

> **警告：** 此操作不可逆，請謹慎使用。建議先指定 `group_ids` 做局部清除，而非清除全部資料。

## 工作流程範例

### 1. 開始新任務

```
1. search_memory_nodes(query="任務相關關鍵字", entity_types=["Preference", "Procedure"])
2. search_memory_facts(query="相關事實")
3. 分析搜索結果
4. 執行任務
5. add_memory_simple(name="任務結果", episode_body="詳細描述")
```

### 2. 儲存用戶偏好

```
用戶：「我喜歡用簡潔的代碼風格」
動作：add_memory_simple(
  name="代碼風格偏好",
  episode_body="用戶偏好使用簡潔的代碼風格，避免冗長的實現",
  group_id="preferences"
)
```

### 3. 記錄操作程序

```
用戶：「部署時先運行測試，然後構建，最後發佈」
動作：add_memory_simple(
  name="部署程序",
  episode_body="部署流程：1. 運行測試 2. 構建應用 3. 發佈到生產環境",
  group_id="procedures"
)
```

### 4. 圍繞特定節點探索

```
# 先找到感興趣的節點
result = search_memory_nodes(query="專案架構")
node_uuid = result[0]["uuid"]

# 圍繞該節點搜索相關事實
facts = search_memory_facts(
  query="相關資訊",
  center_node_uuid=node_uuid
)
```

### 5. 長文本背景處理

```
# 長文本使用背景模式，避免等待
result = add_memory_simple(
  name="會議紀錄",
  episode_body="（一份 2000 字元的會議紀錄...）",
  group_id="meetings",
  background=true
)
task_id = result["task_id"]

# 稍後檢查進度
status = get_memory_task_status(task_id=task_id)
# status.chunks_done / status.chunks_total 可知進度
```

### 6. 刪除過時資訊

```
# 獲取最近的記憶片段
episodes = get_episodes(last_n=20)

# 找到過時的記憶並刪除
delete_episode(uuid="要刪除的記憶UUID")
```

### 7. 健康檢查

```
# 檢查服務狀態
status = get_status()

# 或使用基本連接測試
connection = test_connection()
```

## 注意事項

### 記憶管理
- 記憶內容應該清晰、具體且有意義
- 使用適當的分組 ID 來組織不同類型的記憶（如 `preferences`、`procedures`、`decisions`）
- 定期檢查和更新過時的資訊
- 保持記憶片段的邏輯大小，避免過長或過短
- 使用描述性的名稱來便於後續檢索

### 模式選擇
- **完整模式**（預設）：記憶可被搜尋，但處理較慢（需要 LLM 進行實體提取）
- **安全模式**（`use_safe_mode=true`）：秒速完成，適合不需要搜尋的記錄
- 重要的偏好、程式和事實務必使用完整模式，以確保可被搜尋到

### 效能相關
- 長文本（>800 字元）會自動切分，無需手動分段
- 使用 `background=true` 可避免長時間阻塞
- `excluded_entity_types` 可減少不需要的實體提取，加快處理速度
- 使用 `entity_types` 參數可更精確地過濾搜索結果

### 安全性
- 刪除操作不可逆，請先確認 UUID 正確
- `clear_graph` 清除的資料無法恢復，建議使用 `group_ids` 做局部清除

## 服務端點（HTTP 模式）

使用 HTTP 傳輸模式時，可用以下端點：

| 端點 | 方法 | 說明 |
|------|------|------|
| `/mcp` | POST | MCP 協議主端點 |
| `/` | GET | Web 管理介面 |
| `/health` | GET | 健康檢查（liveness） |
| `/health/ready` | GET | 深度檢查（含 Neo4j 連線驗證） |
| `/api/stats` | GET | 儀表板統計 |
| `/api/groups` | GET | 取得所有 group_id |
| `/api/nodes` | GET | 瀏覽實體節點（分頁） |
| `/api/facts` | GET | 瀏覽事實（分頁） |
| `/api/episodes` | GET | 瀏覽記憶片段（分頁） |
| `/api/search/nodes` | GET | 向量搜尋節點 |
| `/api/search/facts` | GET | 向量搜尋事實 |
| `/api/memory/tasks` | GET | 列出背景任務 |
| `/api/memory/tasks/{id}` | GET | 查詢單一任務 |
| `/api/nodes/{uuid}` | DELETE | 刪除節點 |
| `/api/episodes/{uuid}` | DELETE | 刪除記憶片段 |
| `/api/facts/{uuid}` | DELETE | 刪除事實 |
| `/api/groups/{group_id}` | DELETE | 刪除整個 group |

**健康檢查範例：**
```bash
# 基本 liveness 檢查
curl http://localhost:8000/health

# 深度 readiness 檢查（含 Neo4j 連線）
curl http://localhost:8000/health/ready
```

> **注意**：SSE 模式（`--transport sse`）已不建議使用，請改用 HTTP 模式（`--transport http`）。
